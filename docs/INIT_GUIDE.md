> **参照文档：** BLUEPRINT.md、DEV_PLAN.md、SYSTEM_INVENTORY.md

## Step 1：创建组织架构

**入口：** 系统管理 → 组织管理
**参照：** BLUEPRINT.md 第1.2节「八中心组织架构」

### 1.1 组织层级规则

系统支持四级组织层级：**公司 → 中心 → 部门 → 组**

建议按以下规则编码：

| 层级 | 编码规则 | 示例 |
|:----:|---------|------|
| 公司 | HQ | HQ |
| 中心 | 缩写 | RD、SM、SC |
| 部门 | 中心缩写-编号 | RD-01（产品设计部）|
| 组 | 部门编号-组编号 | RD-01-01（电磁方案组）|

### 2.2 操作步骤

1. 进入 系统管理 → 岗位管理
2. 新增岗位，填写名称、所属组织、编码、描述
3. 按以上列表依次创建

### 2.3 验收标准

- 所有推荐岗位已创建并归属到正确组织
- 岗位编码唯一、有规则

## Step 3：导入员工

**入口：** 系统管理 → 员工管理

### 3.1 操作步骤

1. 进入 系统管理 → 员工管理
2. 新增员工，填写姓名、工号、手机/邮箱
3. 分配岗位（一人一岗或一人多岗）

### 3.2 关联规则

| 场景 | 规则 |
|------|------|
| 一人一岗 | 操作工→操作工岗位 |
| 一人多岗 | 张三同时是打样技术员和设备管理员 |
| 离职 | 禁用员工，不影响历史数据 |

### 3.3 验收标准

- 员工信息已录入，可查看完整花名册
- 员工已分配到对应岗位

---

## Step 4：配置角色权限

**入口：** 系统管理 → 角色权限
**参照：** SYSTEM_INVENTORY.md（权限码清单）

### 4.1 权限模型

岗位 → 角色 → 权限 三级模型

### 4.2 推荐权限分配

| 角色 | 所属岗位 | 权限码 |
|------|---------|--------|
| 打样角色 | 打样技术员 | sampling:order:read, write |
| 研发主管角色 | 研发主管 | sampling:全部 |
| 销售员角色 | 销售员 | crm:customer/quote/order/complaint read+write |
| 采购员角色 | 采购员 | supplier/purchase read+write |
| 仓管员角色 | 仓管员 | warehouse read+write |

### 4.3 操作步骤

1. 创建角色 → 分配权限码 → 绑定到岗位
2. 用非管理员账号登录验证权限生效

### 4.4 验收标准

- 角色已创建，权限已分配
- 非管理员登录只能看到有权限的页面

## Step 5：配置编码规则

**入口：** 系统管理 → 编码规则
**参照：** DEV_PLAN.md Phase 0.2 编码规则配置

### 5.1 推荐编码方案

| 单据类型 | 编码KEY | 格式 | 示例 |
|---------|---------|------|------|
| 打样工单 | SAMPLING_WO | DY-{年份}-{流水号4位} | DY-2026-0001 |
| NPI项目 | NPI_PROJ | NPI-{年份}-{流水号3位} | NPI-2026-001 |
| 产品编码 | PLM_PROD | P-{年份}-{流水号4位} | P-2026-0001 |
| 物料编码 | ERP_MAT | M-{年份}-{流水号4位} | M-2026-0001 |
| BOM编号 | PLM_BOM | BOM-{年份}-{流水号4位} | BOM-2026-0001 |
| 图纸编号 | DRAWING | DWG-{系列}-{流水号3位} | DWG-YJ4820-001 |
| 报价单 | CRM_QUOTE | BJ-{年份}-{流水号4位} | BJ-2026-0001 |
| 销售订单 | CRM_ORDER | SO-{年份}-{流水号4位} | SO-2026-0001 |
| 供应商编码 | SUPPLIER | S-{年份}-{流水号4位} | S-2026-0001 |
| 采购订单 | PURCHASE_PO | PO-{年份}-{流水号4位} | PO-2026-0001 |

### 5.2 操作步骤

1. 进入 系统管理 → 编码规则
2. 新增规则：选择单据类型、填写前缀、年份格式、流水号位数
3. 点击测试生成验证格式
4. 按列表逐条配置

### 5.3 验收标准

- 所有编码规则已配置
- 测试生成格式正确
- 新建单据时自动按规则编号

## Step 6：配置工作流状态

**入口：** 系统管理 → 工作流
**参照：** DEV_PLAN.md Phase 0.2 状态流转配置

### 6.1 推荐状态流转

打样工单：待审批 → 已审批 → 已分配 → 进行中 → 已完成
销售订单：草稿 → 已确认 → 生产中 → 已发货 → 已完成
采购订单：草稿 → 已审批 → 已下单 → 部分到货 → 全部到货 → 已关闭

### 6.2 操作步骤

1. 进入 系统管理 → 工作流
2. 创建状态（如"待审批"）
3. 创建流转（源状态→目标状态）
4. 按模块逐一配置

### 6.3 验收标准

- 各模块状态已创建
- 状态间流转关系已配置
