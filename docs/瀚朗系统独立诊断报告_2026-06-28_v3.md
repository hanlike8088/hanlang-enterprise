   - `crm.controller.ts` 第 10 行：`import { RequirePermission } from '../common/guards/permission.guard'`
   - `npi.controller.ts` 第 7 行：同上
   - `quality.controller.ts` 第 3 行：同上

   - 这些 Controller 确实使用了 `@RequirePermission` 装饰器

   - `main.ts` 中**没有** `app.useGlobalGuards(new PermissionGuard())`
   - 各模块的 providers 中也**没有**提供 PermissionGuard
   - 没有 `APP_GUARD` 注入

3. **装饰器的本质**
   - `permission.guard.ts` 第 20 行：`@RequirePermission` 只是 `SetMetadata(PERMISSIONS_KEY, [...])`
   - 这只是设置了一个 metadata 标记，**没有任何 Guard 来读取和执行检查**

4. **唯一生效的权限检查**
